# plan-feature

Plan a Weavers feature before implementation.

Feature: [FEATURE_NAME]

Process:
1. Inspect relevant files and existing patterns.
2. Identify affected pages/components/forms/data.
3. Identify design source from Stitch/MCP if relevant.
4. Propose implementation steps.
5. List files likely to change.
6. List risks or unclear requirements.
7. Wait for approval before coding.

Do not implement until the plan is approved.
