# Stitch to Production Implementation Skill

Use this skill when converting Stitch/MCP design output into production code.

Goal:
Turn the approved Stitch design into maintainable, responsive, functional frontend code.

## Process

1. Inspect Stitch/MCP design context.
2. Identify repeated sections and components.
3. Map design blocks to reusable components.
4. Preserve spacing, typography, color, imagery, and hierarchy.
5. Implement responsive behavior.
6. Add real links, CTAs, form state, and interactions.
7. Replace placeholder copy with approved Weavers copy.
8. Check accessibility and mobile behavior.
9. Run available checks.

## Design Fidelity Standards

- Preserve premium spacing.
- Preserve royal blue accent discipline.
- Preserve muted palette.
- Preserve editorial image scale.
- Do not turn design into a generic card grid.
- Do not shrink imagery excessively.
- Do not add random colors.
- Do not add unnecessary shadows or gradients.

## Implementation Standards

- Break large sections into components.
- Keep props typed.
- Keep content configurable.
- Avoid one-off inline styles unless project uses them.
- Use existing styling conventions.
- Ensure links and buttons work.

## Output Expected

- Files changed.
- Components created.
- Design deviations, if any.
- Checks run.
- Screens/pages to manually review.
